CLEANING COVID-19 DOH DATA

Inconsistent data

In this project, data records with inconsistent values are omitted from the data drop. Similar to how clinical database systems flag inconsistent data, this project uses query to identify inconsistent data. Queries are conditions that signify conflict between values entered between two or more data features. Here, 6 queries are identified, which are the following:

Query #1:
HealthStatus = 'RECOVERED' AND RemovalType <> 'RECOVERED' OR
HealthStatus = 'DIED' AND RemovalType <> 'DIED'
Query #2:
DateSpecimen > DateResultRelease OR
DateResultRelease > DateRepConf
Query #3:
Sex <> 'MALE' AND PregnantTab IS NULL OR
Sex = 'MALE' AND PregnantTab IS NOT NULL
Query #4:
Age is not within AgeGroup e.g. AGE = 17 AND AgeGroup <> '15 to 19'
Query #5:
HealthStatus = 'DIED' AND DateDied IS NULL
Query #6:
HealthStatus = 'RECOVERED' AND DateRecover IS NULL

In Figure 1, it shows that almost 2/3 of the data records trigger Query #6. This shows how data cleaning is necessary before treating data for analysis. With Query #6, it is possible that the recovery rates calculated will not reflect with quality data and cause effects especially with policy making.

Figure 1. Cleaning of data by eliminating data records triggering queries.

Missing data

The data drop also consists of features related to the residence of the COVID 19 suspects or patients. These features are Region, Province, City, and Barangay. Unfortunately, the data drop contain data records with incomplete residence information, which tracking and tracing COVID19 become more difficult. To show the general scope, Figure 2 shows the number of data records with no, one, two, three, or all four missing residence information.

Majority of these data records miss one data, often the Barangay information, while roughly 4% of the total number of data records have complete residence information. These missing information are usually identified in the ValidationStatus of the data drop, but nevertheless, this shows that the data drop may not be ready for quality geographic analysis of COVID 19 data.

Figure 2. Number of data records with missing zero to all four residence data are graphed, showing that most data records miss one residence information.

Effect

Given these data cleaning processes, it is natural that the number of data records are reduced. This may affect constants calculated to determine the epidicity of COVID-19. Looking at Figure 3, it is seen how the growth of the confirmed cases changes between raw data and quality data. This implies that quality data is a must especially on data analysis that are senstive in dates.

Figure 3. Sample plot of number of total confirmed cases per day shows the difference between unclean and clean data.
